
---

### **一、基础与语言特性**

1. **语言相关问题**（以 Java/Python/Node.js/Go 为例）：

   - 解释 **垃圾回收机制**（如 JVM 分代回收、Go 的三色标记法）。
   - **并发编程**：线程 vs 进程、锁（乐观锁/悲观锁）、死锁避免、协程（Goroutine）。
   - **内存管理**：堆/栈区别、内存泄漏排查。
   - 语言特性：如 Python 的 GIL、Java 的 HashMap 原理、Go 的 channel 实现。
2. **代码与算法**：

   - 手写算法（常考：二分查找、链表、树遍历）。
   - 设计模式（单例、工厂、观察者等）及实际应用场景。
   - 代码优化与重构案例分析。

---

### **二、数据库与存储**

1. **SQL 数据库（如 MySQL/PostgreSQL）**：

   - 索引原理（B+树）、最左前缀匹配、索引失效场景。
   - 事务与隔离级别（ACID、脏读/幻读）。
   - 分库分表策略、主从复制原理。
   - SQL 优化：EXPLAIN 分析慢查询。
2. **NoSQL（如 Redis/MongoDB）**：

   - Redis 数据类型、持久化（RDB/AOF）、缓存穿透/雪崩解决方案。
   - MongoDB 索引机制、副本集与分片。

---

### **三、系统设计与架构**

1. **API 设计**：

   - RESTful 规范、GraphQL 优劣、API 版本管理。
   - 认证与授权（JWT/OAuth2.0、Session vs Token）。
2. **分布式系统**：

   - 分布式锁（Redis/ZooKeeper 实现）。
   - 消息队列（Kafka/RabbitMQ）应用场景与可靠性保证。
   - 微服务架构：服务发现、熔断降级、CAP 理论。
   - 负载均衡策略（轮询/一致性哈希）。

---

### **四、性能与安全**

1. **性能优化**：

   - 数据库查询优化、缓存策略（缓存更新策略、CDN）。
   - 系统瓶颈定位（压测工具、监控指标）。
   - 高并发场景设计（如秒杀系统）。
2. **安全性**：

   - 常见攻击防范（SQL 注入、XSS、CSRF）。
   - HTTPS 原理、数据加密（对称/非对称加密）。

### **五、运维与部署**

1. **DevOps 基础**：

   - Docker 镜像与容器原理、Kubernetes 核心概念。
   - CI/CD 流程设计。
   - 日志收集与监控（ELK/Prometheus）。
2. **网络基础**：

   - TCP/IP 模型、三次握手/四次挥手。
   - HTTP/1.1 vs HTTP/2、WebSocket。

### **准备建议**

1. **深入理解原理**：不仅要会用，还要知道底层机制（如 Redis 为什么快、MySQL 索引为什么用 B+ 树）。
2. **结合实际项目**：用过往项目经验举例说明你如何解决实际问题（如“我用 Redis 缓存热点数据，QPS 提升了 30%”）。
3. **系统设计练习**：参考《System Design Interview》或类似资料，训练结构化思考。
4. **编码练习**：在 LeetCode 上保持手感，尤其关注高频题。

---

### **示例回答框架**

- **问题**：“如何设计一个高并发的抢购系统？”
- **回答结构**：
  1. 分层设计：负载均衡 → 业务服务 → 缓存 → 数据库。
  2. 关键优化：限流（令牌桶）、库存缓存（Redis 预减）、异步下单（消息队列）。
  3. 容灾：服务降级、数据库分库分表。
